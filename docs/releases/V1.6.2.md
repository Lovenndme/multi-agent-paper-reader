## 本次更新

### 修复旧前端残留的升级事故

- 修复更新 Python 后端后仍可能继续加载旧版 `frontend-prototype/dist` 的问题。
- Vite 生产构建现在必须生成 `dist/build-meta.json`，记录与后端一致的项目版本。
- 前端构建缺失、元数据损坏或前后端版本不一致时，根页面会返回明确的 503 修复指引，不再静默显示旧界面。
- `/api/health` 新增 `frontend_version` 和 `frontend_version_match`；发现不一致时 `ok=false`。
- `index.html` 使用 `Cache-Control: no-cache`，避免浏览器长期保留旧入口文件。

### 不可绕过的发行门禁

- Ubuntu、Windows 和 macOS CI 均会先构建前端，再强制校验前后端版本，最后运行完整测试。
- Windows CI 会额外解析 PowerShell 更新脚本，防止对 Windows 用户发布不可执行的升级流程。
- 发行打包器强制要求 Git 标签、`PROJECT_VERSION` 和前端构建元数据三者完全一致；任一不匹配即停止发布。
- 正式发布工作流要求对应版本存在中文发行说明，不再从英文标签自动生成正文。

### 统一安装与更新

- 新增 Windows PowerShell 更新脚本 `scripts/update.ps1`。
- 新增 macOS/Linux 更新脚本 `scripts/update.sh`。
- 脚本会以 fast-forward-only 方式更新干净的 Git 工作区，安装 Python 与 Node.js 依赖，重建前端并执行版本硬校验。
- 更新不会删除未跟踪文件，不会覆盖本机 `.env` 和 `.paper-reader/` 用户数据。

### 完整可运行发行包

- GitHub Release 新增 `Paper-Reader-V1.6.2.zip`，其中包含已构建且版本匹配的 `frontend-prototype/dist`。
- 正式 ZIP 用户在前端校验通过时无需安装 Node.js；更新脚本会直接复用包内的生产构建。
- 同时提供 `Paper-Reader-V1.6.2.zip.sha256`，用于下载完整性校验。
- 打包器只收录 Git 已跟踪文件和验证过的前端构建，明确排除凭据、`.env`、缓存、模型输出、用户论文和本地数据。

## 升级方法

Windows PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\update.ps1
```

macOS 或 Linux：

```bash
bash ./scripts/update.sh
```

脚本完成后必须按提示重启本地服务。如果从 GitHub Release 下载 ZIP，请使用同名 `.sha256` 文件校验后再解压。
