## 本次更新

### 论文定位与完整视觉读取

- Codex 当前论文工具由 8 个扩展为 10 个，新增论文概览、稳定图表索引和显式整页图像读取。
- 模型现在可以先按章节、页码和图表 ID 导航，再读取精确原文或已验证视觉区域。
- 已保存论文会持久化导航 manifest，历史论文恢复后仍保持稳定的章节与图表定位能力。

### 双质量 PDF 渲染

- 模型视觉识别保留 144 DPI 安全预览，在已验证准确率与高分辨率输入一致的前提下避免不必要的资源消耗。
- 用户查看或导出时保留高分辨率图像：原生点阵图保持源像素密度，矢量图和表格按 600 DPI 渲染。
- 渲染前会估算像素和内存占用；超出安全上限时明确拒绝，不再静默降采样。

### 跨平台兼容与验证

- 修复 Windows 对 POSIX 权限位的合成表示造成的 Codex 论文工具上下文误拒绝，同时保留 macOS/Linux 的所有者专用权限检查。
- 后端完整测试通过：177 passed，20 subtests passed。
- Ubuntu、Windows 和 macOS 三平台 GitHub Actions 全部通过，前端生产构建成功。

## 升级提醒

V1.6.1 自动生成的 GitHub 源码包不包含 `frontend-prototype/dist`。从旧版本直接更新源码后，请在项目根目录重新构建前端：

```powershell
npm --prefix frontend-prototype ci
npm --prefix frontend-prototype run build
```

然后重启本地服务，并在浏览器中按 `Ctrl+F5` 强制刷新。如果未重新构建，旧的 `dist` 可能继续显示早期设置界面。
