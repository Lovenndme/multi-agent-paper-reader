# LangMem 记忆架构评测报告

评测日期：2026-07-15

框架：LangMem 0.0.30

真实模型：Qwen `qwen3.7-max`（快速模式）

评测轮数：3 轮

评测脚本：`scripts/evaluate_memory.py`

## 验收结论

当前 LangMem 架构通过本轮正式可用性验收。三轮真实模型结果完全一致：记忆的新增、拒绝临时信息、纠错和遗忘均成功；定向召回全部首位命中，无关查询全部返回空；跨会话、进程重启及 1,000 条消息压力均通过。

这表示它已经适合本项目当前的小型单机论文管理场景。结论不外推为大规模多用户服务的性能承诺；若未来改为服务端集群，应换用 LangGraph 官方列出的持久化 Store。

## 评测隔离

- 所有动态数据写入系统临时目录，没有修改 `.paper-reader` 中的论文、会话或 SQLite 历史。
- API Key 仅通过现有本地配置读取，没有写入脚本、输出或报告。
- 每个行为用例使用独立论文和会话，避免互相污染。
- 每一轮均新建临时数据库和进程内索引。

## 召回质量与延迟

8 条合成长期记忆覆盖注意力架构、消融实验、训练参数、数据集、局限、硬件环境、标题语言和复现入口。每个定向问题最多返回 3 条记忆，并以 0.48 为最低余弦相似度阈值。

| 路径 | 三轮中位观测 | 三轮最差观测 | 是否阻塞回答 |
| --- | ---: | ---: | --- |
| 本地记忆选择 | 4.1 ms | 5.1 ms | 是，但量级可忽略 |
| 单个后台 LangMem 管理用例 | 2.856 s | 5.139 s | 否 |
| 1,000 消息场景召回 | 3.6 ms | 4.3 ms | 是，但量级可忽略 |

平均每个定向查询只返回 1.375 条记忆；天气问题三轮均为 0 条。与旧架构固定取满 5 个主题相比，Prompt 污染显著降低。

## 已知边界

- LangMem 0.0.30 的依赖 `trustcall` 在当前 LangGraph 1.x 下会产生一条上游弃用警告，不影响功能；`requirements.txt` 已限制 LangGraph `<2.0.0`，避免该弃用接口被移除后直接破坏运行。
- 本地 MiniLM 仅用于检索，不负责决定记忆内容；首次下载和加载模型会产生一次冷启动成本。
- 当前 SQLite adapter 面向本地单进程/轻量使用；它不是分布式数据库。
- 本轮真实行为仅验证了 Qwen。其他厂商可复用同一 LangMem 逻辑，但仍需各自可用的 Key、余额和模型权限才能形成真实上游结论。
